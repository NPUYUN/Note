# Git分支操作

---

## 分支概述
在版本控制中，同时推进多个任务时，为每个任务创建单独的分支，进而将自己的开发工作与开发主线分离开来。在开发自己的分支时，不会影响主线的运行。本质为指针的引用。

- 支持同时进行多个功能的开发，提高开发效率
- 各个分支开发时，失败不会对其他分支有影响，删除重新开始即可

## 分支操作
### 查看分支
```bash
git branch -v
```

### 创建分支
```bash
git branch 分支名
```
分支默认继承当前分支的内容

### 切换分支
```bash
git checkout 分支名
```
即切换指针

### 合并分支
```bash
git merge 分支名
```
把指定分支合并到当前分支上，重名文件将被顶替，只会更改当前分支的内容

### 冲突合并
合并分支时，两个分支在同一文件有两种完全不同的修改时，产生代码冲突，须人为决定新代码的内容。
此时当前分支进入合并模式，须手动打开文件，此时会显示对应的修改内容，手动删除修改内容之外的后保存即可


> 保存完成后，仍需添加到暂存区，提交本地库时 ***不需要带文件名*** ，提交后当前分支合并模式解除
> 冲突合并只会更改当前分支的内容

## 总结
Git分支本质就是两种指针
- **HEAD**：指向当前分支
- **分支指针**：指向各自的分支

所有分支操作都是这两种指针的相互切换以及移动
