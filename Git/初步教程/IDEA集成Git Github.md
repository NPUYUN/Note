# IDEA集成Git
## 配置Git忽略文件
### 为什么要忽略他们
与项目的实际功能无关，不参与服务器的部署运行，且忽略掉能够屏蔽IDE工具之间的差距

### 怎么忽略
1. 创建忽略规则文件 **xxx.ignore**，将忽略的文件后缀写入文件即可，这里略过
2. 在.gitconfig文件中引用忽略配置文件
```bash
[core]
    excludesfile = git.ignore位置
```

### 定位Git程序
在IDEA设置中的版本控制中定位Git程序

## 本地库操作
### 初始化本地库
在项目版本控制(Git)选项中选择创建Git仓库

### 提交暂存区
右击相应文件/项目根目录，选择Git添加即可
不要强制添加追踪忽略过的文件

> 红色文件：未提交到暂存区
> 绿色文件：暂存区文件
> 黄色文件：忽略文件
> 白色文件：提交本地库文件
> 蓝色文件：被追踪过但修改过的文件，可以直接提交本地库

### 提交本地库
右击项目根目录，选择Git提交问地库

## 切换版本
### 查看版本
点击左下角版本控制(Git)，查看log(日志)

### 切换版本
在log界面，右击想要去的版本，点击**checkout**(签出)

## 分支使用
### 创建分支
右击项目选择创建Git分支

### 切换分支
点击左下角版本控制(Git)，可以看到所有分支。此时可以选择切换等操作

### 合并分支
点击左下角版本控制(Git)，可以看到所有分支。此时可以选择分支进行合并当前分支

### 冲突合并
发生冲突时，需进行手动处理冲突，此时IDEA会自动提示选择是否删减以及顺序等。冲突处理完成后会自动提交本地库

# IDEA集成Github
和集成Git类似，图形化界面显示较为简单，不再赘述
主要操作仍为 **Push** **Pull** **Clone**
> URL: 链接地址

# IDEA集成Gitee
和集成Github类似，但需要手动安装Gitee插件
Gitee可以直接复制和迁移Github项目，只需新建项目时转到最下角导入已有库，附有**同步**(即强制刷新)的功能

> Gitee推荐使用 **Http** 传输
> Github推荐使用 **SSH** 传输
